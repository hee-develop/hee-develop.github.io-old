    • Next.js를 이용해 Github blog 만들기(1)

    2021-06-07

    React등을 공부하기 위해 Github pages와 Next.js를 이용해 정적 사이트(블로그)를 만들어 보기로 했다.

    1. next.js 초기화

    npx create-next-app my-blog
npm i @types/node @types/react typescript -D
touch tsconfig.json
npm run dev

    우선 create-next-app을 이용해 기본적인 뼈대를 만든다. Typescript를 사용할 예정이므로, 관련 패키지와 설정 파일을 추가해 둔다. tsconfig.json파일을 만들어 둔 채로 next 앱을 실행하면 자동으로 typescript 설정을 대신해 준다.

    Error: Image Optimization using Next.js' default loader is not compatible with next export. Possible solutions: - Use next start to run a server, which includes the Image Optimization API. - Use any provider which supports Image Optimization (like Vercel). - Configure a third-party loader in next.config.js. - Use the loader prop for next/image.

    2. 패키지 설치

    Markdown를 분석하고 HTML문서로 표현하기 위해 remark를 사용한다. Markdown의 헤더 정보를 분석하기 위해 gray-matter를 사용한다. 이 예제에서는 글을 폴더별로 나눌 예정(/articles/글제목/*.*)인데, fs만 가지고 파일 경로를 구하는 것은 상당히 귀찮은 일이었다. 때문에 glob을 이용해 경로를 조금 더 쉽게 구하도록 한다.

    npm i remark remark-html gray-matter -D
npm i glob -D

    3. Markdown 파일 불러오기

    다음은 파일을 실제로 불러 오는 작업을 진행한다. 아직 포스트를 작성하지 않았으므로, 적당히 파일을 하나 생성한 뒤 진행한다.

    ---
title: test
date: '2021-01-01'
---

hello, world!

    대충 이런 느낌의 파일을 만들고, 관리를 쉽게 하기 위해 적당한 폴더에 집어 넣는다. 그런 다음, 해당 폴더를 찾아 폴더 내 포스트를 읽어 오도록 한다.

    export const getPostFile = (fileDir: string) => {
  return fs.readFileSync(fileDir);
};

export const getPostData = (post: Buffer) => {
  const postData = matter(post);
  // gray-matter를 이용해 MD 내부의 데이터를 분리해낸다.
  // data 속성 내부가 헤더 정보가 되며, 예제에선 title과 date를 넣었다.

  return {
    ...postData,
    title: postData.data.title,
    date: postData.data.date,
  };
};

export function getPostPaths() {
  const filePaths = glob.sync('./articles/**/*.md');
  // 포스트를 articles폴더 내에 저장하였다. /**/*.md 는 articles내 모든 폴더 안에 있는 *.md 파일을 의미한다.
  const fileName = filePaths
    .map(fp => fp.match(/\/(\w|-)+\.md$/g)[0])
    .map(res => res.replace(/[\/(.md)]/g, ''));
  // 파일 이름을 id로 사용하고 싶었기 때문에, 일부러 파일 이름은 별도로 빼도록 했다.

  const postPaths: PostPathType[] = [];
  for (let i = 0; i < filePaths.length; i++) {
    postPaths[i] = {
      path: filePaths[i],
      fileName: fileName[i],
    };
  }

  return postPaths;
}

export function getAllPostData() {
  return getPostPaths()
    .map(postPath => ({ fileName: postPath.fileName, file: getPostFile(postPath.path) }))
    .map(({ fileName, file }) => ({ ...getPostData(file), fileName }));
}

    이제 getAllPostData를 부르면, articles폴더 내의 모든 포스트의 정보를 가져올 수 있게 된다. 제대로 동작하는지 확인하기 위해, index에 대충 넣어본다.

    export async function getStaticProps(): GetStaticProps = async () => {
  const posts = getAllPostData().map(postData => {
    return {
      id: postData.fileName,
      date: postData.date,
      title: postData.title,
      content: postData.content,
    }
  });

  console.log(posts);

  return {
    props: {
      posts,
    },
  };
}

    주의: getAllPostData()함수는 트랜스파일 단계에서 불린다. 즉 컴포넌트에서 직접 부르는 것은 불가능하므로, Next의 API(getStaticProps())에서 불러와야만 한다.
